# 11 — Ea Implementation Blueprint

Status: ACTIVE_BLUEPRINT  
Authority: Active-source framework

## Build order
1. Confirm repository, Drive folder and ChatGPT Project names.
2. Upload active-source files to GitHub `active-source/`.
3. Upload the same compact active-source files to the ChatGPT Project.
4. Keep the 89-file phase package in archive only.
5. Review and customize Level 1 files.
6. Insert Project instruction block.
7. Configure private Ea Custom GPT.
8. Connect Gmail, Calendar, Contacts, Drive and web search.
9. Configure scheduled routines if desired.
10. Run Level 1 QA validation.
11. Approve Level 1 release.
12. Only after Level 1 approval, decide whether to plan Level 2.

## Minimum viable Level 1
- Runtime core.
- Source authority/project learning.
- Email/calendar/meeting workflows.
- Due diligence/legal/financial support.
- Templates/registers/indexes framework.
- Project setup.
- QA gate.

## Production-ready Level 1
Minimum viable Level 1 plus:

- real contact/company index;
- active projects index;
- product reference map;
- price reference map;
- application reference map;
- email search query library;
- follow-up tracker;
- confidentiality classification;
- validation evidence.

## Level 2 rule
Level 2 is HOLD until Level 1 is finalized, validated and approved.
