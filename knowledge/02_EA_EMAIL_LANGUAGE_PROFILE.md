# Ea Email Language Profile Requirements

Status: DRAFT  
Priority: Critical

## Purpose

Define the knowledge Ea needs so email drafts sound like Ruben/NTSN rather than generic assistant writing.

## Required profile sections

| Section | Description |
|---|---|
| Default language | When to write English vs Norwegian |
| Tone | Formality, warmth, directness, technical depth |
| Greeting style | Typical openings by recipient type |
| Closing style | Typical endings and sign-offs |
| Sentence style | Short/medium/long sentence tendency |
| Paragraph style | Compact paragraphs, bullets, structure |
| Customer tone | How to write to customers/prospects |
| Supplier tone | How to request info/help from suppliers |
| Partner tone | How to write to distributors/agents |
| Follow-up style | Polite reminders and escalation |
| Delay/apology style | How to acknowledge delays or errors |
| Commercial tone | Pricing, lead time, volume and negotiation wording |
| Technical tone | How much detail to include in technical replies |
| Claim caution | Phrases to avoid overpromising |
| Preferred phrases | Reusable phrases extracted from sent emails |
| Avoided phrases | Phrases Ruben/NTSN should not use |
| Signature choice | Which signature to use by context |

## Evidence needed

Recommended evidence sources:

- sent Gmail messages to customers;
- sent Gmail messages to suppliers;
- sent Gmail messages to partners;
- follow-up emails;
- quotation/proposal emails;
- technical clarification emails;
- Norwegian emails;
- meeting reschedule emails.

## Privacy rule

Do not store private customer details in the style profile. Convert examples into generalized writing rules and phrase patterns.

## Approval rule

The first generated email language profile must be PENDING_REVIEW until Ruben approves it.
