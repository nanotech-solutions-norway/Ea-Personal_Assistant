# Ea Version Date Rule Prompt

Status: ACTIVE_PROMPT_ADDENDUM

Use this addendum together with all Ea knowledge extraction prompts.

```text
Before extracting knowledge, identify the newest actual source available.

Prefer:
1. highest explicit version number, such as v1.4 or v1.43;
2. newest actual creation, edit, update or change date;
3. date inside the document content;
4. folder or repository context;
5. upload or viewed date only as weak evidence.

If the same file exists in several formats, use the most complete and newest actual version unless formats conflict.

Cross-check extracted file knowledge against relevant Gmail correspondence where available.

If correspondence confirms the file, use the latest file as source of truth. If correspondence conflicts with or only partly confirms the file, classify the item as PENDING_REVIEW and ask Ruben for confirmation.

If required knowledge cannot be found in mail, files, links, project sources, NTSN websites, collaborator websites or public sources, ask Ruben to confirm or provide the source of truth.
```

Apply this addendum to product knowledge, email language profile, writing format, templates, company context, client/partner context, pricing/commercial terms, projects/operations and legal/financial boundaries.
