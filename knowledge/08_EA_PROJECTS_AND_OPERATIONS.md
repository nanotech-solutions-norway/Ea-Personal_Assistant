# Ea Projects and Operations

Status: DRAFT  
Priority: High

## Purpose

Define the operational knowledge Ea needs for active projects, domains, repositories, client work and next actions.

## Required project fields

| Field | Description |
|---|---|
| Project name | Canonical project name |
| Purpose | Short project description |
| Status | Active, hold, dormant, closed |
| Current milestone | Current phase/state |
| Owner/contact | Responsible person or team |
| Source locations | GitHub, Drive, website, project folder |
| Repositories/domains | Relevant links or identifiers |
| Related products | Relevant NTSN/Ea/Atlas/SolarEX products |
| Latest validated decision | Most recent approved decision |
| Known blockers | Risks or missing items |
| Next action | Recommended next step |
| Update frequency | Daily, weekly, monthly, as needed |
| Status | DRAFT, PENDING_REVIEW, APPROVED or CANONICAL |
